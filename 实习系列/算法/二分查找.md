# 二分查找

#### 参考文章：https://zhuanlan.zhihu.com/p/79553968

### 1. 基本形式

```cpp
int binarySearch(int[] nums, int target) {
    int left = 0; 
    int right = nums.length - 1;

    while(left <= right) { // 注意
        int mid = left + (right - left) / 2; // 注意
        if(nums[mid] == target)
            return mid; 
        else if (nums[mid] < target)
            left = mid + 1; // 注意
        else if (nums[mid] > target)
            right = mid - 1; // 注意
    }
    return -1;
}
```

- 问题：
  1. 为什么 `while` 中的条件为 `<=` 而不是 `<` ？
     - 因为初始化 `right` 的赋值是 `nums.length - 1`，即最后一个元素的索引，而不是 `nums.length`，所以搜索的范围是 `[left, right]` 是闭区间
     - 比如没找到的话，那就要保证所有有可能的数都被搜索过了，如果用 `<` 那在 `left==right` 时就会退出循环，但是区间是 `[left, right]` 就会导致少搜索了一个 `left`
  
  2. 为什么要 `int mid = left + (right - left) / 2;` ？
     - 怕 `left + right` 溢出
  
  3. 为什么是 `left = mid + 1;` 和 `right = mid - 1;` ？有时候又是 `left = mid` 和 `right = mid`？
     - 如果在确认 `mid` 不是 `target` 的时候，应该搜索 `[left, mid - 1]` 或 `[mid + 1, right]` ，而不应该再重复搜索 `mid`

- 所以也可以根据这个区间的思路写出 `[left, right)` 的二分查找写法：
  ```cpp
  int binarySearch(int[] nums, int target) {
      int left = 0; 
      int right = nums.length; // 注意
  
      while(left < right) { // 注意
          int mid = left + (right - left) / 2;
          if(nums[mid] == target) 
              return mid; 
          else if (nums[mid] < target) 
              left = mid + 1; // 注意
          else if (nums[mid] > target) 
              right = mid; // 注意
      }
      return -1;
  }
  ```

  

### 2. 寻找左侧边界

```cpp
int left_bound(int[] nums, int target) {
    if (nums.length == 0) return -1;
    int left = 0;
    int right = nums.length;

    while (left < right) {
        int mid = (left + right) / 2;
        if (nums[mid] == target)
            right = mid; // 注意
        else if (nums[mid] < target)
            left = mid + 1;
        else if (nums[mid] > target)
            right = mid;
    }
    return left;
}
```

- 为什么这样写可以找到最左侧的边界？
  - 因为有 `if (nums[mid] == target) right = mid;` 保证在找到 `target` 的时候不会立刻返回，而是在 `[left, mid)` 中继续搜索，不断向左收缩            

- 为什么没有返回 -1 的逻辑？如果没找到，返回的 `left` 又是什么含义呢？
  - 返回的 `left` 的含义是小于 `target` 的元素的个数

### 3. 寻找右侧边界

```cpp
int right_bound(int[] nums, int target) {
    if (nums.length == 0) return -1;
    int left = 0, right = nums.length;

    while (left < right) {
        int mid = (left + right) / 2;
        if (nums[mid] == target) {
            left = mid + 1; // 注意
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid;
        }
    }
    return left - 1; // 注意（或 right - 1，一样的）
}
```

- 为什么可以寻找右侧边界？（答案和寻找左侧边界一样）
- 为什么是 `return left - 1; `？
  - 因为我们对 `left` 的更新必须是 `left = mid + 1`，就是说 while 循环结束时，`nums[left]` 一定不等于 `target` 了，而 `nums[left-1]` 可能是 `target`

- 如果 nums 中不存在 target 这个值，怎么办？

  - 类似之前的左侧边界搜索，因为 while 的终止条件是 `left == right`，就是说 `left` 的取值范围是 `[0, nums.length]`，所以可以添加两行代码，正确地返回 -1：
    ```cpp
    while (left < right) {
        // ...
    }
    if (left == 0) return -1;
    return nums[left-1] == target ? (left-1) : -1;
    ```
