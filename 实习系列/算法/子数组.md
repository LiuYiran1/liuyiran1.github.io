子数组（Subarray）题目是算法面试中的高频考点。由于子数组要求**连续**，这使得它在解题思路上有别于“子序列”（Subsequence，不要求连续）。

解决子数组问题，通常有以下几种核心思路和技术：

### 1. 滑动窗口 (Sliding Window)
这是处理子数组最常用的技巧，通常用于解决**“满足某种条件的最长/最短子数组”**或**“固定长度子数组”**的问题。
*   **适用场景**：
    *   数组元素通常为正数（或者具有单调性）。
    *   寻找最小/最大长度。
    *   固定窗口大小（如：计算长度为 $k$ 的所有子数组的和）。
*   **核心逻辑**：
    1.  初始化左右指针 `left = 0, right = 0`。
    2.  不断移动 `right` 扩大窗口，并更新窗口内的状态。
    3.  当满足（或不满足）某个条件时，移动 `left` 缩小窗口。
*   **经典题目**：[长度最小的子数组](https://leetcode.cn/problems/minimum-size-subarray-sum/)、[无重复字符的最长子串](https://leetcode.cn/problems/longest-substring-without-repeating-characters/)。

### 2. 前缀和 + 哈希表 (Prefix Sum + HashMap)
如果题目涉及到**“子数组之和等于 $k$ / 为 $k$ 的倍数”**，且数组中包含**负数**，滑动窗口会失效，此时通常考虑前缀和。
*   **核心公式**：`Sum(i, j) = PrefixSum[j] - PrefixSum[i-1]`。
*   **关键思路**：
    *   将子数组的和转化为两个前缀和的差。
    *   利用哈希表记录 `PrefixSum` 出现的值及其索引（或出现次数）。
    *   寻找 `PrefixSum[curr] - target` 是否在哈希表中。
*   **经典题目**：[和为 K 的子数组](https://leetcode.cn/problems/subarray-sum-equals-k/)、[每个元音包含偶数次的最长子字符串](https://leetcode.cn/problems/find-the-longest-substring-containing-vowels-in-even-counts/)（前缀异或和）。

### 3. 动态规划 (Dynamic Programming)
用于解决**“最值问题”**，特别是当子数组的状态与前一个元素的状态有明确关联时。
*   **典型技巧（以第 $i$ 个元素结尾）**：
    定义 `dp[i]` 为**以第 $i$ 个元素结尾**的满足条件的子数组属性（如最大和）。这样做可以保证“连续性”。
*   **Kadane's Algorithm**：
    这是求解“最大子数组和”的经典 DP 优化算法。
    *   `dp[i] = max(nums[i], dp[i-1] + nums[i])`
*   **经典题目**：[最大子数组和](https://leetcode.cn/problems/maximum-subarray/)、[乘积最大子数组](https://leetcode.cn/problems/maximum-product-subarray/)。

### 4. 单调栈 (Monotonic Stack)
如果问题涉及到子数组内的**“最大值或最小值”**（例如：求所有子数组中最小值之和），通常会用到单调栈。
*   **核心思路**：
    利用单调栈寻找每一个元素作为“区间最小值/最大值”的边界（左边第一个比它小的数，右边第一个比它小的数）。
*   **经典题目**：[子数组的最小值之和](https://leetcode.cn/problems/sum-of-subarray-minimums/)、[柱状图中最大的矩形](https://leetcode.cn/problems/largest-rectangle-in-histogram/)（本质也是子数组问题）。

---

### 如何选择思路？（避坑指南）

| 题目特征                           | 首选方案               |
| :--------------------------------- | :--------------------- |
| **元素全是正数**，求满足条件的长度 | **滑动窗口**           |
| **包含负数**，求子数组和等于 $K$   | **前缀和 + 哈希表**    |
| 求**最大/最小**和、乘积            | **动态规划 (Kadane)**  |
| 涉及子数组内的**最大/最小元素**    | **单调栈**             |
| 涉及**固定长度 $K$** 的属性        | **固定窗口的滑动窗口** |

### 总结口诀
*   **求和用前缀**：看到子数组和，先想前缀和。
*   **正数用滑窗**：全是正数求长度，滑动窗口最快。
*   **最值用 DP**：看到最大子数组和，Kadane 走起。
*   **极值用单调栈**：涉及子数组里的最大/最小值，考虑单调栈找左右边界。
