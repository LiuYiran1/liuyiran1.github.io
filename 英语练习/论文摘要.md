> Automated program repair (APR) techniques have demonstrated the potential to  reduce the debugging burden significantly. However, existing repair  tools often generate plausible patches that fail to address the  underlying root cause, leading to the risk of regression errors. In this paper, we propose a novel approach that leverages large language models to capture the semantic intent of code changes. By integrating symbolic execution with neural models, our framework can verify the correctness  of generated patches more effectively than traditional test-based  validation. Experimental results on benchmark datasets show that our  method improves the repair accuracy by 15% compared to state-of-the-art  techniques.

- debugging burden：调试负担
- plausible patches：似是而非的补丁
- Experimental results：实验结果
- leverage：借助
- Benchmark datasets：基准数据集

> Test suite prioritization (TSP) aims to improve the effectiveness of  regression testing by reordering test cases to expose faults sooner.  Despite the proliferation of various techniques, there is still a lack  of sufficient research regarding the performance of these techniques in  the context of deep learning-based software. In this paper, we conduct  an empirical study on the applicability of traditional TSP strategies  for deep learning systems. We observe that conventional metrics, such as code coverage, are inadequate to represent the structural behavior of  neural networks. Consequently, we propose a novel metric based on neuron coverage to complement the existing strategies. Our extensive experiments demonstrate that our method significantly outperformsies  existing state-of-the-art approaches in both fault detection capability  and execution time efficiency.

*   **Test suite prioritization (TSP)**：测试套件优先级排序
*   **Regression testing**：回归测试
*   **Code coverage**：代码覆盖率
*   **Neuron coverage**：神经元覆盖率（深度学习测试中的关键概念）
*   **Fault detection capability**：错误检测能力
*   **Metric**：度量指标（务必区分于 Matrix）
*   **A aims to improve B by C**：A 旨在通过 C 来提升 B。（例：TSP 旨在通过重排测试用例来提升回归测试效率。）
*   **Despite the proliferation of X, there is a lack of Y**：尽管 X 已经层出不穷/繁多，但在 Y 方面仍存在不足。（这是论文开头“引出研究动机”的标准句式。）
*   **A is inadequate to represent B**：A 不足以描述/代表 B。（用于反驳传统方法的局限性。）
*   **A outperforms B in C**：A 在 C 方面优于 B。（用于描述实验结果的对比。）

> Large language models (LLMs) have shown remarkable capabilities in generating code snippets to assist developers. However, the prevalence of  hallucinated APIs and insecure coding patterns in LLM-generated code  poses significant risks to software security. Existing approaches mainly rely on post-generation filtering or static analysis to mitigate these  issues. In this paper, we introduce a reinforcement learning-based  framework that integrates security-aware feedback loops into the code  generation process. By rewarding the model during training based on  security-relevant metrics, our approach encourages the model to generate safer code without compromising its functionality. Extensive  evaluations on multiple benchmarks demonstrate that our proposed method  significantly reduces the injection of vulnerable patterns while  maintaining high code quality.

*   **Hallucinated APIs**：幻觉 API（指模型凭空捏造不存在的函数库）。
*   **Insecure coding patterns**：不安全的代码模式（指容易引发漏洞的代码写法）。
*   **Post-generation filtering**：生成后筛选。
*   **Security-aware feedback loops**：安全感知的反馈循环。
*   **injection**：注射、引入
*   **Vulnerable patterns**：脆弱模式（常指带有安全漏洞的代码片段）。
*   **Compromise**：妥协/折衷（常用于 trade-off 场景，如 "without compromising performance"）。
*   **The prevalence of X poses significant risks to Y**：X 的普遍存在对 Y 构成了显著风险。
*   **Existing approaches mainly rely on A to mitigate B**：现有方法主要依赖 A 来缓解 B。（mitigate 是学术论文缓解风险/问题的标准动词）。
*   **Integrate A into B**：将 A 集成到 B 中。
*   **Without compromising X**：在不损害/牺牲 X 的前提下。
*   **Maintain high X**：保持高质量的 X。
